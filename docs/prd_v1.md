# 집들이 초대 홈페이지 MVP PRD

> 작성일: 2026-06-19 · 기반 문서: proposal.md (요구사항 0~7)
> 입주 예정: 2027년 10월 · 호스트: 도토리

---

## 핵심 정보

**목적**: 호스트 도토리의 내집마련 집들이 행사를 소개하고, 초대 손님이 집들이를 선택해 참여 신청·관리·방명록 작성을 한 곳에서 처리할 수 있도록 한다.
**사용자**: 도토리가 초대한 지인(카카오 계정 보유자) 및 호스트 도토리(관리자)

---

## 사용자 여정

```
1. 메인 페이지 (비로그인)
   - 배너 동영상 + 집들이 소개 섹션 + "집들이 참여하기" CTA 버튼 확인

   ↓ CTA 버튼 클릭 or 내비게이션 "집들이 참여하기" 탭 클릭

2. 집들이 목록 페이지 (비로그인 열람 가능)
   - 집들이 카드 목록(대표 이미지·명·편성·일시) 열람

   ↓ 카드 이미지 클릭 or 카드 내 "참여하기" 버튼 클릭

   [비로그인] → 로그인 페이지 이동 → 카카오 OAuth → /auth/callback → 집들이 상세 복귀
   [로그인]   → 집들이 상세 페이지 직접 진입

3. 집들이 상세 페이지 (로그인 필수)
   - 상세 정보 확인(이미지·명·편성·일시·드레스코드·비고)

   ↓

   [미참여 상태] → "참여하기" 버튼 클릭 → 참여 완료 → 참여자 목록 갱신
   [이미 다른 집들이 참여 중] → "변경하기" 버튼 클릭 → 기존 참여 해제 + 새 참여 원자 처리
   [현재 집들이 참여 중] → "취소하기" 버튼 클릭 → 참여 레코드 삭제

   ↓ 참여 완료 후

   참여자 목록(닉네임 + 프로필 이미지) 전체 표시 (페이지네이션 없음)

4. 방명록 페이지 (로그인 필수)
   - "도토리에게 하고싶은 말" 메시지 작성(로그인 필수)
   - 작성 후: 최신글 상단 기준 시간순 목록 표시(내용·작성시간만, 익명)

5. 관리자 페이지 (관리자 전용)
   - 집들이 CRUD (등록·수정·삭제)
   - 환경변수 ADMIN_KAKAO_IDS 허용 목록으로 접근 제어
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | 카카오 OAuth 로그인 | 카카오 계정으로 로그인, 닉네임·프로필 이미지 저장/표시, 로그아웃 제공 | 모든 참여·방명록 기능의 전제 조건, 방문자 식별 수단 | 로그인 페이지, 인증 콜백 페이지, 헤더 공통 |
| **F002** | 메인 배너 동영상 | banner.mp4 자동재생·무음·루프·playsInline, 모바일 대응 | 호스트 요구사항 1번, 집들이 분위기 전달 핵심 요소 | 메인 페이지 |
| **F003** | 집들이 소개 섹션 | 메인 페이지 내 행사 전반 소개 텍스트/이미지 표시 | 방문자가 행사 개요를 파악하는 기본 콘텐츠 | 메인 페이지 |
| **F004** | 집들이 목록 조회 | 집들이 카드 목록 표시 (대표 이미지·명·편성·KST 일시), 비로그인 열람 허용 | 방문자가 참여할 집들이를 선택하는 출발점 | 집들이 목록 페이지 |
| **F005** | 집들이 상세 조회 | 상세 정보 표시 (대표 이미지·명·편성·KST 일시·드레스코드·비고), 로그인 필수 | 참여 전 필수 정보 확인, 비로그인 차단 후 로그인 유도 | 집들이 상세 페이지 |
| **F006** | 참여 신청/취소/변경 | 1인 1집들이 참여(DB UNIQUE 보장), 취소(레코드 삭제), 다른 집들이로 변경(원자 처리), 정원 무제한 | 서비스 핵심 비즈니스 로직, 요구사항 6·6.1 전체 충족 | 집들이 상세 페이지 |
| **F007** | 참여자 목록 표시 | 참여자 닉네임·프로필 이미지 전체 표시 (페이지네이션 없음), 로그인 후 상세 페이지 내 표시 | 방문자가 동행자를 확인하는 기능, 요구사항 6 충족 | 집들이 상세 페이지 |
| **F008** | 방명록 작성 및 열람 | "도토리에게 하고싶은 말" 메시지 작성 (로그인 필수), 화면에는 익명(내용·시간만), DB에는 user_id 저장, 최신글 상단 정렬 | 요구사항 7·7.1 전체 충족, 호스트-손님 소통 창구 | 방명록 페이지 |
| **F009** | 집들이 관리 (CRUD) | 관리자 전용 집들이 등록·수정·삭제, 환경변수 허용 목록으로 관리자 식별 | 호스트가 행사 정보를 직접 관리하는 운영 필수 기능 | 관리자 페이지 |

### 집들이 상세 페이지

> **구현 기능:** `F005`, `F006`, `F007`, `F011` | **인증:** 필수 (비로그인 차단)

| 항목 | 내용 |
|------|------|
| **역할** | 특정 집들이의 전체 정보를 표시하고, 참여 신청·취소·변경 및 참여자 목록을 제공하는 핵심 페이지 |
| **진입 경로** | 집들이 목록 페이지에서 카드 클릭 (비로그인 시 로그인 페이지 → OAuth 완료 후 자동 복귀) |
| **사용자 행동** | 집들이 상세 정보 확인, 참여 신청/취소/변경, 참여자 목록 확인 |
| **주요 기능** | • 상세 정보 표시: 대표 이미지, 집들이명, 편성, KST 일시(`YYYY.MM.DD (요일) HH:mm KST`), 드레스코드, 비고<br>• 참여 상태에 따른 버튼 분기:<br>&nbsp;&nbsp;&nbsp;&nbsp;- 미참여: **"참여하기"** 버튼 → 참여 레코드 INSERT<br>&nbsp;&nbsp;&nbsp;&nbsp;- 현재 집들이 참여 중: **"취소하기"** 버튼 → 참여 레코드 DELETE<br>&nbsp;&nbsp;&nbsp;&nbsp;- 다른 집들이 참여 중: **"변경하기"** 버튼 → 기존 DELETE + 신규 INSERT 원자 처리<br>• 1인 1집들이 DB UNIQUE 제약(`participations.user_id`) 보장<br>• 참여자 목록 전체 표시: 닉네임 + 프로필 이미지 (페이지네이션 없음) |
| **다음 이동** | 참여/취소/변경 성공 → 현재 페이지 참여자 목록 갱신, 실패 → 에러 메시지 표시 |

| **진입 경로** | 헤더 "관리자" 메뉴 클릭 (관리자 계정 로그인 시에만 메뉴 표시), 직접 URL 접근 (비관리자 차단) |
| **사용자 행동** | 집들이 등록·수정·삭제, 등록된 집들이 목록 확인 |
| **주요 기능** | • 집들이 목록 테이블 표시 (관리용)<br>• **"집들이 등록"** 버튼 → 등록 폼 (이미지·명·편성·일시·드레스코드·비고·상세설명 입력)<br>• 집들이별 **"수정"** 버튼 → 수정 폼<br>• 집들이별 **"삭제"** 버튼 → 확인 후 삭제<br>• 이미지 업로드: Supabase Storage 활용<br>• 환경변수 `ADMIN_KAKAO_IDS`와 로그인 계정 kakao_id 서버사이드 대조 (미들웨어 차단) |
| **다음 이동** | 등록/수정 성공 → 관리자 페이지 목록 갱신, 삭제 성공 → 목록에서 제거, 권한 없음 → 메인 페이지 리디렉션 |

## 데이터 모델

### profiles (사용자 프로필)
| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | Supabase auth.users.id 참조 | UUID (PK) |
| kakao_id | 카카오 고유 식별자 (관리자 판별에 사용) | text |
| nickname | 카카오 닉네임 (화면 표시명) | text |
| profile_image_url | 카카오 프로필 이미지 URL | text |
| created_at | 최초 가입 시각 | timestamptz |

### housewarmings (집들이 이벤트)
| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID (PK) |
| name | 집들이명 (예: 그린나래) | text |
| organization | 편성 (예: 21대 총학생회) | text |
| event_at | 행사 일시 (UTC 저장, KST 표시) | timestamptz |
| image_url | 대표 이미지 URL (Supabase Storage) | text |
| dress_code | 드레스코드 (예: 할로윈) | text |
| note | 비고 (예: 부부동반) | text |
| description | 상세 설명 (선택) | text (nullable) |
| created_at | 등록 시각 | timestamptz |

### participations (참여 기록)
| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID (PK) |
| user_id | 참여자 식별자 — UNIQUE 제약으로 1인 1집들이 보장 | UUID → profiles.id |
| housewarming_id | 참여 집들이 | UUID → housewarmings.id |
| created_at | 참여 신청 시각 | timestamptz |

> UNIQUE(user_id) 제약 필수 — 취소는 레코드 DELETE, 변경은 housewarming_id UPDATE (또는 DELETE+INSERT 원자 처리)

### guestbook (방명록)
| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID (PK) |
| user_id | 작성자 추적용 (화면 미노출, DB 추적 전용) | UUID → profiles.id |
| content | 메시지 내용 | text |
| created_at | 작성 시각 (최신순 정렬 기준) | timestamptz |

> 화면 조회 시 content, created_at만 SELECT (user_id 제외). 관리자 추적 쿼리는 profiles JOIN으로 nickname·kakao_id 확인 가능.

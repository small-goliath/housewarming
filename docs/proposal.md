## 1. 프로젝트 개요

내집마련 기념 입주 후 **집들이 초대 및 참여 관리 홈페이지**를 구축한다.
방문자는 카카오 로그인 후 여러 집들이 중 하나에 참여 신청할 수 있고, 호스트(도토리)에게
익명으로 메시지를 남길 수 있다.

| 항목 | 내용 |
| --- | --- |
| 서비스명 | 집들이 초대 홈페이지 (가칭) |
| 핵심 사용자 | 호스트의 지인/초대 손님 |
| 호스트(관리자) | 도토리(사이트 소유자) |
| 목표 | 집들이 소개·참여 신청·참여자 확인·방명록을 한 곳에서 제공 |

## 4. 기능 요구사항 (필수)

### 4.0 카카오 로그인 *(요구사항 0)*
- 카카오 OAuth 로그인 제공.
- 로그인 시 **닉네임 + 프로필 이미지**를 저장/표시하여 "누가 참여했는지" 식별 가능.
- 로그인 상태는 전역(헤더 등)에서 확인 가능하며 로그아웃 제공.

### 4.1 메인 페이지 - 배너 동영상 *(요구사항 1)*
- 메인 상단에 `banner.mp4` 배너 동영상 노출.
- 자동재생/무음/루프(`autoplay muted loop playsInline`) 권장, 모바일 대응.

### 4.2 집들이 소개 *(요구사항 2)*
- 메인 페이지 내 집들이 행사 전반 소개 섹션 노출(텍스트/이미지).

### 4.3 집들이 참여하기 버튼 · 탭 *(요구사항 3)*
- 메인에 "집들이 참여하기" CTA 버튼 노출.
- 전역 내비게이션에 "집들이 참여하기" 탭 제공.

### 4.4 집들이 목록 페이지 *(요구사항 4)*
- 여러 집들이를 카드 목록으로 노출. 각 카드에 다음 표시:
  - **대표 이미지**
  - **집들이명** (예: 그린나래)
  - **편성** (예: 21대 총학생회)
  - **일시 (KST)**
- 각 카드에 "참여하기" 버튼 노출.
- 비로그인 상태에서도 목록 열람은 가능(상세 진입부터 로그인 필수).

### 4.5 집들이 상세 페이지 *(요구사항 5)*
- 진입 경로: 카드의 **대표 이미지** 또는 **참여하기 버튼** 클릭.
- **상세 진입부터 카카오 로그인 필수** *(5.2)*. 비로그인 시 로그인 페이지로 유도 후 복귀.
- 상세 표시 항목 *(5.1)*:
  - 대표 이미지
  - 집들이명 (예: 그린나래)
  - 편성 (예: 21대 총학생회)
  - 일시 (KST)
  - 드레스 코드 (예: 할로윈)
  - 비고 (예: 부부동반)

### 4.6 참여 · 참여자 목록 *(요구사항 6)*
- 상세 페이지에서 참여 신청 가능.
- 참여 시 **본인이 참여한 집들이의 참여자 목록** 노출(닉네임 + 프로필 이미지), **페이지네이션 없음**.
- **1인 1집들이만 참여 가능** *(6.1)* — DB 유니크 제약으로 보장.
- 참여 **취소** 및 **다른 집들이로 변경** 모두 허용.
  - 취소: 참여 레코드 삭제.
  - 변경: 기존 참여를 해제하고 새 집들이로 재신청(원자적 처리).
- **참여 정원 제한 없음**.

### 4.7 방명록 페이지 *(요구사항 7)*
- 페이지명/문구: **"도토리에게 하고싶은 말"**.
- 작성에 **카카오 로그인 필수**.
- 화면에는 **완전 익명**으로 표시(작성자 정보 미노출, 내용·작성시간만 표시).
- **DB에는 작성자(user_id)를 저장**하여 "누가 무엇을 작성했는지" 쿼리로 조회 가능 *(요구사항 7)*.
- 작성란 **아래**에 다른 사용자 글을 **시간 순으로 리스트업** *(7.1)* — 정렬: **최신글이 상단**(내림차순).

## 6. 데이터베이스 스키마 (PostgreSQL / Supabase)

> 모든 시각은 `timestamptz`로 저장(UTC). 화면 표기는 **KST(Asia/Seoul)** 로 변환.

### 6.1 `profiles` — 사용자 프로필
| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| `id` | uuid (PK) | Supabase `auth.users.id` 참조 |
| `kakao_id` | text | 카카오 고유 식별자 |
| `nickname` | text | 카카오 닉네임(표시명) |
| `profile_image_url` | text | 프로필 이미지 URL |
| `created_at` | timestamptz | 가입 시각 |

### 6.2 `housewarmings` — 집들이(이벤트)
| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| `id` | uuid (PK) | |
| `name` | text | 집들이명 (예: 그린나래) |
| `organization` | text | 편성 (예: 21대 총학생회) |
| `event_at` | timestamptz | 일시(KST 변환 표기) |
| `image_url` | text | 대표 이미지 |
| `dress_code` | text | 드레스 코드 (예: 할로윈) |
| `note` | text | 비고 (예: 부부동반) |
| `description` | text (nullable) | 상세 설명(선택) |
| `created_at` | timestamptz | |

### 6.3 `participations` — 참여
| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| `id` | uuid (PK) | |
| `user_id` | uuid (FK→profiles.id) | **UNIQUE** → 1인 1집들이 보장 |
| `housewarming_id` | uuid (FK→housewarmings.id) | |
| `created_at` | timestamptz | |

- `UNIQUE(user_id)` 제약으로 6.1 요구사항 보장.
- 취소 = 레코드 삭제, 변경 = `housewarming_id` 갱신(또는 삭제 후 재삽입).

### 6.4 `guestbook` — 방명록("도토리에게 하고싶은 말")
| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| `id` | uuid (PK) | |
| `user_id` | uuid (FK→profiles.id) | **작성자 추적용(화면 미노출)** |
| `content` | text | 메시지 내용 |
| `created_at` | timestamptz | 작성 시각(정렬 기준) |

- 화면 조회 쿼리는 `content`, `created_at`만 select → 익명 표시.
- 관리자 추적 쿼리 예시:
  ```sql
  SELECT g.created_at, p.nickname, p.kakao_id, g.content
  FROM guestbook g
  JOIN profiles p ON p.id = g.user_id
  ORDER BY g.created_at DESC;
  ```

### 6.5 보안(RLS) 가이드
- `guestbook`: INSERT는 로그인 사용자만, SELECT는 작성자 컬럼 비노출 형태로 제공(또는 view 사용). 작성자 직접 조회는 서버/관리자 전용.
- `participations`: 본인 레코드만 변경/삭제 가능.
- `housewarmings`: SELECT는 전체 허용, INSERT/UPDATE/DELETE는 관리자만.

## 7. 인증 플로우 (카카오)

1. 사용자가 로그인 버튼 클릭 → Supabase Auth 카카오 OAuth 시작.
2. 카카오 동의 후 `/auth/callback`으로 복귀, 세션 발급.
3. 최초 로그인 시 `profiles`에 닉네임·프로필 이미지 upsert.
4. 보호 경로(`/housewarmings/[id]`, `/guestbook`, `/admin`)는 미들웨어에서 세션 검사.
5. `/admin`은 추가로 환경변수 허용 목록과 대조.

## 8. 디자인 가이드

- **무드**: 모던 다크 + 포인트 컬러(베리어웃 톤). 동영상 배너가 돋보이는 히어로 구성.
- 다크 배경 위 카드/타이포 대비 확보, 포인트 컬러로 CTA·강조.
- 반응형(모바일 우선), 카드형 목록, 부드러운 전환/호버.
- 한국어 UI, 일시는 항상 `YYYY.MM.DD (요일) HH:mm KST` 형식 권장.
